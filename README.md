# Syracuse Low-Stress Bike Router

A static web app that finds the least-stressful cycling route between two points in Syracuse, NY. Routing runs entirely in the browser using a Rust navigation engine compiled to WebAssembly — no server required.

**Live:** [dev.edwarddeaver.me/syr-bike-navigation](https://dev.edwarddeaver.me/syr-bike-navigation)

---

## Features

- **Low-stress routing** — Dijkstra's algorithm weighted by Level of Traffic Stress (LTS), strongly preferring paths, trails, and protected bike lanes over unprotected roads
- **Hill avoidance** — optional penalty that routes around grades above 4%, with near-prohibitive cost above 12%
- **Stress overlay** — full street network coloured by LTS level (green / yellow / red)
- **Turn-by-turn directions** — step list with bearing-based turn detection; tap any step to pan the map
- **Route stats** — distance, estimated time (10 mph), LTS breakdown, max grade
- **Address search** — geocoding via Photon/Komoot with keyboard navigation
- **Export to PDF** — print the route and directions via browser print dialog
- **Mobile-friendly** — full-screen map with a slide-out drawer on small screens

## Stack

| Layer | Technology |
|-------|-----------|
| Routing engine | Rust → WebAssembly (`wasm-pack`, `wasm-bindgen`) |
| Map | Leaflet 1.9 + CartoDB Voyager tiles |
| Geocoding | Photon (forward) + Nominatim (reverse) |
| Data pipeline | Python 3.13 + DuckDB |
| Elevation | OpenTopoData SRTM30m API |
| Street data | PeopleForBikes BNA, January 2025 |

## Getting Started

### Prerequisites

- [Rust](https://rustup.rs/) + `wasm-pack` (`cargo install wasm-pack`)
- [Python 3.13+](https://python.org) + [uv](https://github.com/astral-sh/uv)

### 1. Build the WASM module

```bash
cd nav_wasm
wasm-pack build --target web --release
```

Output lands in `pkg/`.

### 2. Run the data pipeline

Only needed if you want to regenerate `syr_map.json` (e.g. after changing the source data or re-fetching elevations).

```bash
cd db_loader
uv sync
uv run python load.py            # parse graph → DuckDB
uv run python fetch_elevations.py  # fetch SRTM elevations (takes ~3 min)
uv run python export_json.py     # export DuckDB → syr_map.json
```

### 3. Serve locally

Any static file server works:

```bash
python -m http.server 8080
# open http://localhost:8080
```

## Data

Street network and LTS classifications from [PeopleForBikes BNA](https://bna.peopleforbikes.org/) (January 2025).
Elevation from [OpenTopoData](https://www.opentopodata.org/) SRTM30m via the public API.

LTS scale: **1** = comfortable (trails, paths, calm streets) · **2** = moderate (bike lanes) · **3** = stressful (no infrastructure)

Routing cost: `length × LTS_penalty × hill_penalty` where LTS penalties are 1×, 2×, 10× and hill penalties range from 1× (≤4%) to 50× (>12%).
